# SEEKR Local Retention Policy

This policy covers local internal-alpha runs. It does not authorize cloud sync or aircraft command upload.

## Local Data Classes

- `data/` is the default persistent local store for developer runs.
- `.tmp/rehearsal-data/` is the recommended field-laptop store for clean rehearsals.
- `.tmp/playwright-data/` and other `.tmp/*-test-*` directories are disposable test stores.
- `.tmp/overnight/` stores watchdog logs and status for unattended verification.
- `.tmp/release-evidence/` stores field-laptop install-integrity JSON, SHA-256, and Markdown checksum evidence, including repository-safety files such as `.gitignore` and `.npmrc`.
- `.tmp/rehearsal-evidence/` stores read-only API snapshots captured before/after local field-laptop rehearsals.
- `.tmp/rehearsal-start-smoke/` stores bounded local operator-start smoke evidence with exact ordered rows for setup, doctor, API, client, source-health, readiness, and clean shutdown checks.
- `.tmp/rehearsal-notes/` stores fill-in operator rehearsal notes, completed closeouts, and their JSON manifests.
- `.tmp/safety-evidence/` stores command-boundary static-scan evidence for local acceptance and readiness reviews.
- `.tmp/api-probe/` stores final local API probe evidence that `/api/session` read back acceptance release checksum and command-boundary scan summaries after acceptance recording.
- `.tmp/completion-audit/` stores local alpha completion audits that separate completed software evidence from real-world hardware blockers.
- `.tmp/demo-readiness/` stores internal demo/audit handoff packages with false hardware claims and next-evidence guidance.
- `.tmp/bench-evidence-packet/` stores operator task-card packets for collecting missing real-world bench evidence.
- `.tmp/handoff-index/` stores final local handoff indexes plus digest verification reports that verify the latest evidence pointers agree and record linked-artifact SHA-256 digests without clearing real-world blockers.
- `.gstack/qa-reports/` stores local gstack browser QA Markdown reports plus named screenshots generated by `npm run qa:gstack`; these are browser/API QA evidence only and do not validate hardware.
- `.tmp/source-control-handoff/` stores read-only source-control handoff audits for package/README GitHub references, the parent GitHub landing README fresh-clone path into `SEEKR/software`, local Git metadata, configured remotes, GitHub remote refs/default branch, shallow fresh-clone contents for the landing README, package manifest, lockfile, env template, rehearsal start wrapper, operator quickstart, and `npm ci --dry-run` install consistency, local HEAD versus GitHub default branch SHA, machine-readable `workingTreeClean` plus clean-worktree status-line evidence, persisted blocked/warning check counts, and manual publication next-action checklists without initializing Git, committing, pushing, or validating hardware.
- `.tmp/handoff-bundles/` stores copied review bundles built only from verified handoff indexes, their linked local artifacts, the latest gstack workflow-status artifact, any workflow-referenced local gstack browser QA report and named screenshots, the latest TODO audit artifact, the latest source-control handoff artifact, the latest plug-and-play setup artifact, the latest local AI prepare artifact, the latest operator-start plug-and-play doctor artifact, the latest exact-row rehearsal-start smoke artifact, the strict local AI smoke status, and the operator quickstart, plus copied-bundle verification reports with digest, copied strict-AI smoke exact scenario/loopback-Ollama-URL/validator/unsafe-text/mutation checks, copied-workflow exact row order/QA screenshot/TODO exact blocker-category order/top-level-count/support-shape/source-control/setup/local-AI-prepare/doctor/exact-row smoke repository-safety, startup, and source-control handoff semantic checks, operator-start doctor-to-acceptance freshness, operator-start doctor source-control evidence matching against the packaged source-control handoff, copied operator quickstart strict-AI-smoke/advisory-AI semantic checks, and high-confidence secret-scan coverage results.
- `.tmp/plug-and-play-setup/` stores non-destructive local setup evidence for `.env` creation/preservation, project-local rehearsal data preparation, package-script coverage, and disabled command-upload/hardware-actuation flags.
- `.tmp/plug-and-play-doctor/` stores local startup preflight evidence for package scripts, package `engines`/`packageManager`, Node/package-lock readiness, `.npmrc` `engine-strict`, `.gitignore` protection for local secrets/generated artifacts including runtime `data/`, latest source-control handoff artifact semantics for the GitHub reference, local `tsx`/`concurrently`/`vite` dependency binaries needed by `npm run dev`, project-local env setup, local Ollama model availability, API/client port availability or healthy already-running SEEKR listeners, read-only listener PID/cwd diagnostics for occupied non-SEEKR ports when available, local data directory safety, and disabled command-upload/hardware-actuation environment flags. Occupied default ports pass when they already serve a healthy local SEEKR instance, or when the start wrapper can prove unconfigured defaults will be auto-replaced with free local API/client ports; explicitly configured occupied ports stay warnings. The start wrapper normalizes `PORT`/`SEEKR_API_PORT` and auto-selects free local API/client ports when unconfigured defaults are busy, while explicit operator port choices remain explicit. Final review bundles use the latest `profile: "operator-start"` doctor, so rerun `npm run doctor` after any source-control refresh or bounded rehearsal-start smoke before packaging.
- `.tmp/plug-and-play-readiness/` stores local plug-and-play readiness audits that prove the app, local AI, API, QA, setup artifact, local AI prepare artifact, operator-start doctor preflight, rehearsal-start smoke artifact, source-control handoff artifact semantics, review bundle, safety boundary, and persisted real-world blocker count/list are internally consistent while preserving real-world blockers.
- `.tmp/goal-audit/` stores final prompt-to-artifact objective audits that map named requirements to evidence while preserving real-world blockers and a persisted remaining-blocker count/list.

## Files To Preserve After A Rehearsal

- `mission-events.ndjson`: append-only mission event log and hash-chain audit record.
- `latest-state.json`: last local GCS snapshot for operator recovery context.
- `replays/*.json`: exported replay manifests with event log, final state hash, evidence index, adapter metadata, and redacted session/config export metadata.
- Operator notes that record the final hash shown by `/api/verify`, `/api/readiness`, report, and incident log.
- `.tmp/rehearsal-evidence/seekr-rehearsal-evidence-*.{json,md}` files captured before and after the run.
- `.tmp/rehearsal-start-smoke/seekr-rehearsal-start-smoke-*.{json,md}` files when the operator-start path is used as local startup evidence.
- `.tmp/rehearsal-notes/seekr-rehearsal-note-*.{json,md}` templates and `.tmp/rehearsal-notes/seekr-rehearsal-closeout-*.{json,md}` files with filled operator timestamps, replay id, final hash, shutdown note, and deviations.
- `.tmp/safety-evidence/seekr-command-boundary-scan-*.{json,md}` files for the accepted build.
- `.tmp/api-probe/seekr-api-probe-*.{json,md}` files for the final accepted API surface readback.
- `.tmp/completion-audit/seekr-completion-audit-*.{json,md}` files when the run is used for demo/readiness review.
- `.tmp/demo-readiness/seekr-demo-readiness-*.{json,md}` files when the run is handed to an internal reviewer.
- `.tmp/bench-evidence-packet/seekr-bench-evidence-packet-*.{json,md}` files when a real bench run is being planned from the demo package.
- `.tmp/handoff-index/seekr-handoff-index-*.{json,md}` files when a handoff package is sent for internal review.
- `.tmp/handoff-index/seekr-handoff-verification-*.{json,md}` files after verifying a handoff package digest table.
- `.gstack/qa-reports/seekr-qa-*.md` and referenced `.gstack/qa-reports/screenshots/*.png` files when the workflow artifact or review bundle points at browser QA evidence.
- `.tmp/source-control-handoff/seekr-source-control-handoff-*.{json,md}` files when GitHub publication or handoff state is reviewed, including the manual next-action checklist for any missing landing guidance, incomplete fresh-clone contents, local Git metadata, origin, remote refs, unpublished HEAD, dirty worktree, and re-audit step.
- `.tmp/handoff-bundles/seekr-handoff-bundle-*` directories and matching `.json`/`.md` manifests when a copied review bundle is sent for internal review.
- `.tmp/handoff-bundles/seekr-review-bundle-verification-*.{json,md}` files after verifying a copied review bundle's digests, copied strict local AI smoke status including loopback Ollama URL, copied gstack workflow semantics, copied QA report/screenshot inclusion, copied source-control handoff semantics, copied plug-and-play setup semantics, copied local AI prepare semantics, copied plug-and-play doctor source-control/repository-safety semantics, matching source-control handoff evidence, freshness, copied operator quickstart setup/ai:prepare/source-control audit/start/strict-AI-smoke/advisory-AI/API/source-health/safety guidance including no AI-created command payloads or validation bypass, and high-confidence secret-scan coverage.
- `.tmp/plug-and-play-setup/seekr-local-setup-*.{json,md}` files when non-destructive local setup evidence is reviewed.
- `.tmp/plug-and-play-doctor/seekr-plug-and-play-doctor-*.{json,md}` files when local startup preflight is reviewed.
- `.tmp/plug-and-play-readiness/seekr-plug-and-play-readiness-*.{json,md}` files when local plug-and-play readiness is reviewed.
- `.tmp/goal-audit/seekr-goal-audit-*.{json,md}` files when the handoff is reviewed against the original objective.

## Evidence References

Evidence assets and spatial assets are stored as metadata references in V1. URIs may point to local thumbnails, frames, logs, maps, reports, spatial previews, or external lab storage. The replay manifest preserves the URI, hash, retention policy, redaction state, and metadata reference; it does not guarantee binary media is embedded.

Before deleting referenced media, export the mission package and confirm the evidence owner has either copied the binary asset elsewhere or marked the reference as intentionally unavailable in the rehearsal notes.

## Suggested Retention

- Keep rehearsal `mission-events.ndjson`, `latest-state.json`, and `replays/*.json` for at least 30 days.
- Keep evidence assets marked `evidence` with the matching replay manifest until the investigation owner releases them.
- Keep `legal-hold` references and binaries until explicitly cleared by the owner.
- Delete `.tmp/playwright-data/` freely; Playwright recreates it.
- Delete `.tmp/overnight/` after the corresponding status has been copied into engineering notes.
- Keep `.tmp/release-evidence/` files with the field-laptop install notes for the matching build.
- Keep `.tmp/rehearsal-evidence/` snapshots with the matching operator notes; they are local API evidence only, not hardware validation.
- Keep `.tmp/rehearsal-notes/` with the matching evidence snapshots. A generated note is only a template until `npm run rehearsal:closeout` validates the required fields and writes a completed closeout.
- Keep `.tmp/safety-evidence/` files with the matching release checksum and acceptance status.
- Keep `.tmp/api-probe/` files with the matching acceptance status; completion audits and handoff indexes rely on them for final session-visible evidence.
- Keep `.tmp/completion-audit/` files with readiness review notes; they are useful for proving which blockers were still real-world-only at the time of review.
- Keep `.tmp/demo-readiness/` files with the matching readiness review notes and completion audit.
- Keep `.tmp/bench-evidence-packet/` with the real bench run plan; it is guidance only and is not a substitute for the evidence artifacts it names.
- Keep `.tmp/handoff-index/` with the demo package and bench packet it verifies; it is only a pointer consistency and digest index plus verification report.
- Keep `.tmp/handoff-bundles/` with the matching handoff index; it is only a copied local review packet and is not physical hardware validation.
- Keep `.tmp/goal-audit/` with the handoff packet it references; it is the final prompt-to-artifact checklist and is not physical hardware validation.

## Redaction And Secrets

Session/config metadata in exports is redacted by design. Do not manually add `SEEKR_INTERNAL_TOKEN`, local credentials, or private URLs to incident notes, replay metadata, or docs. If a token appears in a captured artifact, rotate it and mark that artifact as restricted.

## Verification Before Archive

Run these before archiving a rehearsal directory:

```bash
npm run check
npm run rehearsal:note -- --label archive-check
npm run rehearsal:evidence -- --label archive-check
npm run audit:completion
npm run audit:todo
npm run smoke:rehearsal:start
npm run doctor
npm run handoff:bundle
npm run handoff:bundle:verify
npm run audit:plug-and-play
npm run audit:goal
curl -s http://127.0.0.1:8787/api/verify
curl -s http://127.0.0.1:8787/api/readiness
curl -s http://127.0.0.1:8787/api/replays
```

The final archive should include at least one replay manifest and a matching final state hash in verify, readiness, report, and incident log.
